{# Componente reutilizável de ícone para a sidebar. USO (Jinja — uso estático): {% from "_icon.html" import icon %} {{ icon("folder") }} {{ icon("briefcase", color="#ff4d00") }} {{ icon("users", size=14) }} USO (Alpine — uso dinâmico em x-for de gestão): Para items vindos do banco/estado do Alpine, renderiza via template condicional. O Alpine define `isLucideName(v)` no escopo principal: FALLBACK INTELIGENTE: Se `name` é ASCII puro (ex: "folder", "chart-bar"), assume nome Lucide e renderiza . Se contém caracteres não-ASCII (emoji), renderiza como texto literal. Permite que o template novo entre no ar ANTES da migração do banco (período de transição sem janela quebrada). COR: Default `#a0a4ab` (cinza neutro). Injetada via custom property `--sb-icon-color`, não via inline `color:`, pra permitir override no :hover/:active via CSS sem brigar com specificity. ACESSIBILIDADE: aria-hidden="true" em todos — o texto-label ao lado já descreve o item. RENDERIZAÇÃO: Após inserção no DOM, lucide.createIcons() troca cada por um {%- else -%} {{ name }} {%- endif -%} {%- endmacro %}
